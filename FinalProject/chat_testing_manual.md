# 채팅 시스템 점검 체크리스트 (수동 테스트 매뉴얼)

이 문서는 채팅 시스템의 모든 기능을 순서대로 검증하기 위한 매뉴얼입니다.
**User A(주 테스트 계정)**와 **User B(보조 계정)** 두 개의 계정이 필요합니다.
브라우저를 2개 띄우거나, 시크릿 모드를 활용하여 동시에 로그인해 주세요.

---

## 1. 로그인 및 전역 설정 (Login & Setup)

- [ ] **비로그인 접근 차단**
    - [ ] 로그아웃 상태에서 채팅 탭 클릭 -> "로그인이 필요합니다" 모달 확인.
- [ ] **로그인 (User A, User B)**
    - [ ] User A: 메인 브라우저 로그인.
    - [ ] User B: 시크릿 모드 또는 다른 브라우저 로그인.
- [ ] **프로필 설정 변경 (실시간 미반영 확인)**
    - [ ] **(A)** 채팅 목록 헤더 우측 프로필 이미지 클릭 -> 이미지 변경.
    - [ ] F5를 눌러도 해당 프로필이 유지되는지 확인

## 2. 1:1 채팅 연결 시나리오 (Connection Flow)

### 2-1. 사용자 검색 및 방 생성
- [ ] **(A) 존재하지 않는 유저 검색**
    - [ ] `+` 버튼 -> 닉네임 입력란에 없는 닉네임 입력 -> 엔터.
    - [ ] "해당 사용자를 찾을 수 없습니다" 알림 모달 확인.
    - [ ] `Enter` 또는 `ESC`로 모달 닫기 -> **포커스가 다시 검색창으로 오는지** 확인.
- [ ] **(A) 유저 검색 및 자동 생성**
    - [ ] User B 닉네임 입력 -> 엔터.
    - [ ] 방이 생성됨과 동시에 **즉시 해당 채팅방 내부로 입장**하는지 확인.

### 2-2. 초대 수신 및 거절 (실시간)
- [ ] **(B) 실시간 초대 수신**
    - [ ] 채팅 목록 "초대받은 채팅방" 섹션에 실시간으로 방 노출 확인.
    - [ ] 종 알림 배지 및 알림 내역 클릭 시 채팅 탭으로 이동 확인.
- [ ] **(B) 초대 거절**
    - [ ] "초대받은 채팅방" 목록에서 `❌` 클릭 -> 즉시 삭제 확인.
    - [ ] **(A)** 채팅 채널 내에 **"User B님이 초대를 거절했습니다"** 메시지가 실시간 갱신되는지 확인.
    - [ ] **(A)** 거절 확인 후 채팅방 나가기 (목록에서 사라짐 확인).

### 2-3. 재초대 및 수락
- [ ] **(A) 재초대**
    - [ ] 다시 User B를 검색하여 1:1 채팅방 생성.
- [ ] **(A) 전송 제한 로직**
    - [ ] **B가 수락하기 전**에 메시지 입력 후 전송 시도.
    - [ ] 전송 실패 알림 또는 에러 메시지 확인.
- [ ] **(B) 초대 수락**
    - [ ] `✅` 클릭 -> 입장이 완료되며 **해당 채팅 영역으로 화면이 전환**되는지 확인.
    - [ ] **(A)** 채팅 채널 상대가 들어왔다는 메시지 뜨는지 확인

## 3. 메시지 대화 및 입력 창 제어 (Messaging)

- [ ] **실시간 메시지 송수신**
    - [ ] **(A)** 메시지 입력 -> 엔터 -> (B) 새로고침 없이 수신 확인.
    - [ ] **(B)** 방 외부(목록 등)에 있을 때 알림 배지(숫자) 및 종 알림 생성 확인.
    - [ ] **(B)** 방 입장 시 목록 배지가 사라지는지 확인.
- [ ] **이미지 전송**
    - [ ] **(A)** 입력창 왼쪽 `+` 아이콘 -> 이미지 파일 선택.
    - [ ] (보너스) 너무 큰 파일 선택 시 용량 제한 에러 모달 확인.
    - [ ] 전송 후 미리보기가 뜨는지 확인 -> (B) 실시간 수신 및 이미지 클릭하여 다운로드 확인.
- [ ] **메시지 삭제 (Owner/Author)**
    - [ ] **(A)** 본인 메시지 우클릭 -> `삭제` -> (B) 실시간으로 "삭제된 메시지입니다"로 변경되는지 확인.

## 4. 리액션, 답장, 공지 (Interaction)

- [ ] **공감(리액션) 기능 세부 테스트**
    - [ ] **(A)** 특정 메시지 우클릭 -> 공감 이모지 클릭 -> 숫자 1 표시 확인.
    - [ ] **(B)** 실시간으로 동일 이모지가 뜨는지 확인 -> 버튼 클릭하여 **숫자 2로 올라가는지** 확인.
    - [ ] **(B)** 본인이 누른 공감 버튼 다시 클릭 -> **공감 취소 및 숫자 1 감소** 확인.
    - [ ] **(A)** 이미 누른 공감을 다른 이모지로 변경 시도 -> 실시간으로 아이콘이 바뀌는지 확인.
- [ ] **답장(Reply) 기능**
    - [ ] **(B)** A의 메시지 우클릭 -> `답장`.
    - [ ] 입력창 상단 "User A님에게 답장..." 툴바 노출 확인.
    - [ ] 답장 전송 -> 원본 메시지가 상단에 포함된 말풍선 UI 확인.
    - [ ] **답장 인용구 클릭** -> 원본 메시지가 있는 위치로 스크롤 이동 및 **배경 하이라이트** 확인.
- [ ] **공지(Notice) 기능**
    - [ ] **(A)** 메시지 우클릭 -> `📢 공지 등록`.
    - [ ] 상단에 고정된 공지 배너 생성 확인 -> (B) 실시간 배너 노출 확인.
    - [ ] 스크롤을 멀리 보낸 뒤 **공지 배너 클릭** -> 해당 공지 메시지 위치로 즉시 스크롤 이동 확인.
    - [ ] **(A)** 공지 배너 우측 `X` 버튼 클릭 -> 공지 해제 -> (B) 실시간 배너 사라짐 확인.

## 5. 그룹 채팅방 관리 (Group Management)

- [ ] **그룹 채팅방 생성**
    - [ ] `+` 버튼 -> 그룹 채팅 탭 선택.
    - [ ] 방 제목 15자 이상 입력 시도 -> **글자수 제한** 확인.
    - [ ] 방 제목 입력 및 생성 -> 즉시 입장 확인.
- [ ] **방 정보 변경 (방장)**
    - [ ] **(A)** `설정` -> `방 설정` 탭.
    - [ ] 방 제목 수정 -> (B) 상단 헤더의 방 제목 실시간 변경 확인.
    - [ ] 대표 이미지 변경 -> (B) 실시간 반영 확인.
- [ ] **멤버 초대 및 퇴장**
    - [ ] (본인 닉네임 검색) 참여중인 멤버 검색 결과 확인.
    - [ ] **(A)** `설정` -> `멤버` -> User B 검색 및 `초대` 버튼 클릭.
    - [ ] **(B)** 채팅 리스트 화면에서 해당 채팅방 생기는 것 확인
    - [ ] B 수락 대기중 오른쪽에 x 버튼 클릭 -> 초대 취소
    - [ ] **(B)** 채팅 리스트 화면에서 해당 채팅방 사라짐 확인
    - [ ] **(B)** "초대받은 채팅방" 수락 후 입장.
    - [ ] **(A)** 멤버 목록에서 User B가 "참여 멤버"로 상태 변경 확인.
- [ ] **1대1 채팅방 나가기 (Exit)**
    - [ ] **(B)** `설정` -> `채팅방 나가기` -> 목록에서 즉시 사라짐 확인.
    - [ ] **(A)** 채팅방 내에 "User B님이 퇴장했습니다" 메시지 실시간 확인.
    - [ ] **(A)** 후속 테스트를 위해 다시 초대
    - [ ] **(B)** 후속 테스트를 위해 초대 수락
    - [ ] **(A)** (A가 방장일때) `설정` -> `채팅방 나가기` -> 위임후 나가라는 모달 뜨는지 확인

## 6. 권한 위임 및 강퇴 (Delegate & Kick)

- [ ] **방장 권한 위임**
    - [ ] **(A)** `설정` -> `멤버` 탭 -> User B 옆 `위임` 버튼.
    - [ ] (A) 화면에서 방장 권한(방장 아이콘 및 설정 메뉴 편집권) 사라짐 확인.
    - [ ] **(B)** 본인에게 방장 아이콘 생김 확인 및 **방 설정 탭의 편집 기능 활성화** 확인.
- [ ] **강퇴(Kick)**
    - [ ] **(B)** 방장이 된 B가 `멤버` 탭에서 User A `강퇴`.
    - [ ] **(A)** "방장에 의해 강퇴당했습니다" 알림 모달 노출 확인.
    - [ ] **(A)** 확인 클릭 시 자동으로 채팅 목록 화면으로 리다이렉트 되는지 확인.

## 7. 스크롤 및 검색 심화 (Discovery)

*대화 테스트를 통해 메시지가 30~50개 이상 쌓인 상태에서 진행*

- [ ] **최근 읽은 위치 (Unread Marker)**
    - [ ] **(B)** 채팅방 밖으로 나감.
    - [ ] **(A)** 메시지 15개 이상 도배.
    - [ ] **(B)** 채팅방 재진입 -> 상단에 **"마지막으로 읽은 메시지 위치로 이동"** 버튼 노출 확인.
    - [ ] 버튼 클릭 시 실제 읽지 않은 시작 지점으로 스크롤 점프 확인.
- [ ] **무한 스크롤 (Infinite Scroll)**
    - [ ] **(A)** 채팅 30개 이상인 채팅방에서 나가기
    - [ ] 다시 입장 후 스크롤을 맨 위로 이동.
    - [ ] 30개 추가 로드 확인.
    - [ ] 로드 시 **현재 보고 있던 스크롤 위치가 흔들리지 않고 고정**되는지 확인.
- [ ] **검색 기능 (Search Integration)**
    - [ ] **(A)** 채팅 30개 이상인 채팅방에서 나갔다 다시 들어오기
    - [ ] 검색 아이콘 클릭 -> "ㅋ" 등 자주 쓴 단어 검색.(10번 이상)
    - [ ] **페이징 확인**: 결과가 많을 때 `1 / 10+` 등의 카운터 확인.
    - [ ] **이동/로드 확인**: `▶ (과거)` 버튼 클릭하여 현재 DOM에 없는 과거 메시지로 이동 시, **자동으로 메시지를 페칭하며 해당 위치로 점프**하는지 확인.
- [ ] **새 메시지 알림 (Toast)**
    - [ ] (A),(B) 둘다 스크롤을 위쪽에 둔 상태(가장 아래 채팅을 보고있지 않은 상태) 
    - [ ] (A)가 메시지 전송.
    - [ ] 하단에 플로팅 토스트 확인
    - [ ] (B) 화면에서도 해당 플로팅 토스트 뜨는지 확인
    - [ ] (B) 가 그대로 답장하기
    - [ ] 해당 메시지도 플로팅 토스트로 뜨는지 확인
    - [ ] 토스트 클릭 -> 맨 아래(최신)로 즉시 이동 확인.

## 8. 채팅 목록 및 마이페이지 연동 (UX/UI)

- [ ] **상대방 프로필 모달 및 1:1 연결**
    - [ ] 채팅방 내 상대방 이름/아바타 클릭 -> 프로필 모달 확인.
    - [ ] `1:1 채팅하기` 버튼 클릭 -> **기존에 있던 1:1 방으로 즉시 이동**하는지 확인.
- [ ] **최근 대화방 상단 이동**
    - [ ] 목록 하단에 있던 채팅방에서 메시지를 받거나 보낼 시, 해당 방이 **목록의 최상단으로 끌어올려지는지** 확인.
- [ ] **즐겨찾기 (Pin)**
    - [ ] 목록의 채팅방 우측 별표 아이콘 클릭 (또는 메뉴).
    - [ ] 해당 채팅방이 목록 상단 "즐겨찾기" 섹션에 고정되는지 확인.
- [ ] **종합 카운터**
    - [ ] 여러 방에 안 읽은 메시지가 있을 때, 전체 탭의 숫자 배지가 정확히 합산되어 표시되는지 확인.
